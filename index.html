JEPSO-CI Hook

Once you've configured your repository correctly, you'll need to trigger the tests so JEPSO-CI knows to run them. You can do this manually, or have it done automatically every time you push to GitHub

Manually Queue Tests

To test a repo; just put the User, Repo and Tag you wish tested in these fields and click "Queue Test". You can also check whether JEPSO-CI is correctly configured by clicking "Test Configuration" which won't actually run any tests.

Automaticaly Queue Tests

You can set up GitHub with this url as a post-receive hook and JEPSO-CI will automatically re-test your library every time you push changes. To do this, go to your GitHub page and click on settings, you then need to add https://hook.jepso-ci.com as a WebHook URL. It should then look like:
